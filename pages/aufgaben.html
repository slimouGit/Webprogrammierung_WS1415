Aufgaben

Beschreibungen der Einsendeaufgaben aus dem Seminar Web-Programmierung.

  • ESA1 - HTML

    Diese Aufgabe ist die Grundlage für alle folgenden Aufgaben. Sie werden alle noch kommenden ESA's hier einbauen. Wenn Sie möchten, dürfen diese Aufgabe(n) zu Ihrem Abschlußprojekt ausbauen. Erstellen Sie eine Webpräsenz zu einem Thema Ihrer Wahl in XHTML 1.0 strict oder HTML 5 wie folgt:

    • Die Indexseite. Sie enthält ein Willkommen für den Anwender mit Informationen zur gesamten Webpräsenz.
    • Eine Seite mit Informationen zu Ihrer Person und Ihren (Vor)kenntnissen.
    • Eine oder mehr Seiten zu dem von Ihnen gewählten Thema.
    • Eine Seite mit Verlinkungen zu den Themen CSS, HTML, semantisch korrrektes HTML, JavaScript, Validierung, Webstandards, Accessibility, Usability, XML. Geben Sie zu jedem Link eine kurze schriftliche Beschreibung.

    Jedes einzelne HTML-Dokument besteht aus:

    • Logobereich. Dieser Bereich muß nicht zwangsweise mit einer Grafik ausgestattet werden, eine Überschrift ist ausreichend.
    • Inhaltsbereich. Hier gibt es schriftlichen, möglicherweise auch grafischen Inhalt für die jeweilige Seite.
    • Navigationsbereich.
    • Fußzeile, die vielleicht auf ein Impressum o.ä. verweist. Dieser Bereich muß sich nicht zwangsweise am Seitenende befinden.

    Planen Sie bereits jetzt, daß Sie ein Layout für den Browser, und eines für den Druck und eines für ein Handheldgerät erstellen werden. Legen Sie sich eine übersichtliche Verzeichnisstruktur an, benennen Sie Ihre Dateien sinnvoll. "ESA1.html" ist kein sinnvoller Dateiname.

    In den Metagangaben stehen Ihre Namen und Matrikelnummern. Alle Dateien sind valide und semantisch korrekt. Wenn Sie von diesen Vorgaben aus bestimmten Gründen abweichen wollen, dann lassen Sie mich das vorab wissen.

    Semantisch korrekt bedeutet: Verwenden Sie nur solche Elemente, die Ihre Struktur beschreiben. Und verwenden Sie nur die Elemente, die Sie wirklich benötigen. Verwenden Sie auf gar keinen Fall Tabellen, Überschriften und andere Elemente für die Gestaltung. Die Gestaltung interessiert erst ab ESA2.

  • ESA 2 - CSS

    Umsetzung des geplanten Layouts mit CSS: Verwenden Sie Ihre ESA1 und beginnen Sie mit der Gestaltung.

    • Versuchen Sie das Layout mit Hilfe von CSS-Dateien umzusetzen. Erstellen Sie mindestens zwei CSS-Dateien, eine für den Screen und eine für Print. Im Autorenstylesheet für den Screen verwenden Sie Media-Queries, um eine abgespeckte Version für ein mobiles Gerät zu erzeugen. Möglicherweise möchten Sie die verschiedenen Styles auch noch anderweitig in verschiedenen Dateien ordnen (z.B. Grundsätzliches, Schriften, Positinierung, …). Vorrang hat das Stylesheet für den Browser, die anderen braucht nur „angedeutet“ zu sein.
    • Verwenden Sie eine Reset-Datei oder eine Normalize-Datei und begründen Sie Ihre Entscheidung im Kommentar.
    • Binden Sie CSS-Regeln wie folgt ein: im "head" mit dem "link"-Element über andere CSS-Dateien (@import) mit höchster Priorität im html (inline-style) Seien Sie sich über die verschiedenen Funktionen und Gewichtigkeiten der Einbindungsmöglichkeiten im Klaren.
    • Verwenden Sie Ihre Selektoren sinnvoll, versuchen Sie es auch mit Nachfahren- und Attributselektoren.
    • Verwenden Sie mindestens ein Pseudo-Element oder eine Pseudo-Klasse, zusätzlich zu üblichen Linkzuständen, wie hover etc.

    Alle Dateien, auch die CSS-Dateien müssen valide sein. Warnungen für die CSS-Dateien sind bei vorliegender sinnvoller Begründung, im Kommentar, erlaubt.

    Empfangen Sie den Anwender und bieten Sie ihm ein Spiel an. Eine Bildergeschichte mit mehreren Ausgängen, wobei der Anwender selbst seinen Weg wählen kann. Ein Schnickschnackschnuck, ein Schiebepuzzle, ein Memory o.ä. Sie dürfen sich gern auch selbst eine js-Idee überlegen, es muß kein Spiel sein. Seien Sie innovativ :-)

    Vorgefertigte Lösungen aus dem Netz werden nicht bewertet. Sollten Sie einzelne passende Codeblöcke im Netz finden, die Ihnen helfen, so dürfen Sie diese gern verwenden, sofern Sie alles entfernen, was Sie nicht benötigen und den Code verstehen und erklären können. Bedenken Sie außerdem, daß Ihre HTML-Dateien hinterher noch valide sein müssen, was mit vorgefertigten Lösungen nicht immer der Fall ist.

    Halten Sie das JavaScript vom HTML getrennt und binden Sie es "unobstrusive" ein. Halten Sie sich grundsätzlich an bereits gelernte Programmierkonzepte. Deklarieren Sie möglichst lokal, teilen Sie auf in Funktionen, verwenden Sie Argumente und Rückgabewerte,

  • ESA 4 - XML

    Erstellen Sie zu Ihrer Seite eine Sitemap in XML und eine dazugehörige Schema-Datei. Beide Dateien müssen valide und in Ihre Webpräsenz sinnvoll eingebunden sein.

    Verstehen Sie Ihre Schema-Datei und das Zusammenspiel von xsd und xml.
